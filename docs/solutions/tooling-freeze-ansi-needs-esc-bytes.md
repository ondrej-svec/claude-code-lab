# freeze fixture renders escape codes as text instead of colors

## Symptom

You hand-author a `.ansi` fixture under `scripts/capture/sessions/` and run
`./scripts/capture/capture-cli.sh <stem>`. freeze either:

- Errors with `Language Unknown · specify a language with the --language flag`, OR
- Runs with `--language ansi` and writes a PNG that shows your escape sequences (`[38;5;73m`, `[0m`, etc.) as visible text instead of applying colors.

## Cause

`.ansi` files must contain real `0x1b` (ESC) bytes before each `[…m` sequence. If you write the file with literal `[` characters where ESC should be (because your editor doesn't insert ESC, or your `Write` tool serializes the string as plain text), freeze sees no ANSI to interpret. With no language hint, freeze guesses by extension and bails. With `--language ansi`, freeze syntax-highlights the `.ansi` *as source code*, which renders the escape codes as visible text.

## Fix

Write the fixture via something that emits real ESC bytes. The repo's other fixtures are generated by `scripts/capture/generate-fixtures.py`, which uses Python f-strings with `"\x1b"`:

```python
ESC = "\x1b"
def f(code, text):
    return f"{ESC}[{code}m{text}{ESC}[0m"

content = f("38;5;73", "Hello")  # bytes: \x1b[38;5;73mHello\x1b[0m
Path("session.ansi").write_text(content, encoding="utf-8")
```

Then `./scripts/capture/capture-cli.sh <stem>` works with no `--language` flag.

## How to verify

```bash
head -c 20 scripts/capture/sessions/<stem>.ansi | xxd
```

The first byte should be `1b` (ESC), not `5b` (`[`). Existing fixtures show:

```
00000000: 1b5b 3338 3b35 3b37 356d ...   # GOOD: starts with ESC + [
```

If you see:

```
00000000: 5b33 383b 353b 3735 6d ...      # BAD: starts with literal [
```

your file is missing ESC bytes.

## What to look for

- A new `.ansi` fixture that doesn't render styled output
- A fixture authored by hand-writing escape sequences in a code editor (the `Write` tool included)
- freeze warning about unknown language, OR successful render that shows codes as text
