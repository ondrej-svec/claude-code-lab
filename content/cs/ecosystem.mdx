# Ekosystém

Claude Code sám o sobě je dobrý agent. S ekosystémem se stává platformou, kterou rozšiřuješ, místo aby ses na něco čekal.

Tahle kapitola je rozhlídnutí, ne tutoriál. Projdi, označ, co tě zajímá, vrať se pro hloubku později.

## Skilly

**Skill** je balík instrukcí + volitelných skriptů, které si Claude načte podle potřeby. Doménový playbook — bydlí v `~/.claude/skills/`, aktivuje se sám, když situace sedí.

**Příklad, který stojí za prozkoumání:** [`visualize`](https://github.com/ondrej-svec/heart-of-gold-toolkit). Vezme libovolný markdown — plán, brainstorm, architekturu — a udělá z něj uhlazený, sdílitelný HTML artefakt. Jedna dobře ohraničená úloha, dobře napsané instrukce, pár pomocných skriptů. Tvar dobrého skillu.

### Postav si vlastní

Tři kroky:

1. **Vyber si jednu konkrétní úlohu**, kterou děláš opakovaně. Ne „pomoz mi s kódem" — třeba *„převeď tenhle plán na sdílitelný HTML artefakt"* nebo *„napiš release note naším hlasem."*
2. **Napiš `SKILL.md`** s krátkým frontmatter blokem a jasnými instrukcemi:

   ```md
   ---
   name: muj-skill
   description: Jedna věta — kdy to má Claude aktivovat?
   ---

   # Můj skill

   Když tě někdo požádá o X, udělej tohle:

   1. Přečti vstup.
   2. Aplikuj Y.
   3. Vrať Z.
   ```

3. **Dej to do `~/.claude/skills/muj-skill/`.** Claude to najde příště.

Začni hrubě. Použij dvakrát. Dolaď. To je ta smyčka.

→ [Skills docs](https://docs.claude.com/en/docs/claude-code/skills)

## Pluginy

**Plugin** balí související skilly, příkazy a hooky do jednoho instalovatelného celku. Nainstaluj jednou, získáš celý workflow.

Šestá kapitola provází instalací a používáním [Compound Engineering pluginu](https://github.com/every-to/compound-engineering) — Averyho smyčky brainstorm → plan → work → review → compound. Ten si zkus jako první. [Super Powers](https://github.com/iannuttall/claude-super-powers) je další, o kterém má smysl vědět — širší pick-and-choose toolkit.

Jako reference: [Heart of Gold toolkit](https://github.com/ondrej-svec/heart-of-gold-toolkit) je malá kolekce skillů, které jsem si zabalil pro svoji každodenní práci (`visualize` je jeden z nich). Ne něco, co bys teď potřeboval instalovat — příklad toho, co vznikne, jakmile napíšeš tři čtyři skilly, o které nechceš přijít. Jeden takový si postavíš sám.

→ [Plugins docs](https://docs.claude.com/en/docs/claude-code/plugins)

## MCPs

**Model Context Protocol** umožňuje Claudovi mluvit s externími službami — filesystem, databáze, prohlížeče, API — jako s prvotřídními tooly.

Jeden, který vyčnívá pro webové vývojáře:

- **Chrome DevTools MCP** — Claude dostává živý přístup ke Chrome console, network, performance profileru. "Otevři stránku, řekni mi, proč je LCP 3,2 s, oprav to." Claude se teď může opravdu podívat.

Další MCPs existují pro Gmail, Google Drive, Linear, Notion, Postgres a další. Instaluj to, co se hodí tvé práci.

→ [MCP docs](https://docs.claude.com/en/docs/agents-and-tools/mcp)

## Subagenti

Když Claude spustí úkol, který by spálil moc kontextu — hluboký průzkum codebase, velký review — použije **subagenta**. Subagent pracuje ve vlastním kontextovém okně a vrátí ti shrnutí.

Subagenty obvykle nevoláš přímo. Jen si všimneš, že hlavní konverzace zůstává čistá, i když na pozadí běží těžký úkol.

Proč to má význam: tvůj kontextový prostor je konečný. Subagenti ho drží zaměřený na to, na čem ti záleží.

## Remote a mobil

Dvě cesty, jak spustit Claude, když nejsi u svého stolu:

- **Remote control** — Claude Code běží na tvé pracovní stanici, ovládáš ho z prohlížeče nebo telefonu. Užitečné pro dlouhotrvající joby, které odstartuješ z meetingu.
- **Mobilní aplikace** — lehčí, pro rychlé review, čtení historie konverzace, schvalování čekajících úkolů.

Přihlas se stejným účtem a pokračuj uprostřed práce.

Ekosystém je nabídka, ne povinnost. Instaluj, co ti skutečně ušetří čas. Zbytek může počkat.
