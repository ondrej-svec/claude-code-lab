# Než začneme

Tohle není prohlídka funkcí. Je to praxe. Nainstaluješ Claude, pustíš ho na ukázkový projekt, budeš ladit na reálných změnách a odejdeš si s postupem, který použiješ i zítra. Žádné předvádění věcí, které pak stejně nepotřebuješ.

Celý průvodce — kapitoly, ukázkové projekty, doprovodný skill — žije na **[github.com/ondrej-svec/claude-code-lab](https://github.com/ondrej-svec/claude-code-lab)**. Naklonuj si ho a máš všechno lokálně. Oficiální dokumentaci Anthropicu ke Claude Code najdeš na [code.claude.com/docs/en/overview](https://code.claude.com/docs/en/overview) — to je zdroj pravdy, kdykoliv někam odkazujeme.

## Instalace

Dvě cesty. Začni s desktop aplikací — k prvnímu funkčnímu spuštění se dostaneš nejrychleji.

### Claude desktop — pro tenhle lab doporučená

Stáhni z [claude.ai/download](https://claude.ai/download). Desktop aplikace má Claude Code jako integrované prostředí. Žádné nastavování terminálu, žádná konfigurace. Přihlas se a otevři složku projektu.

### Claude Code CLI — až budeš chtít větší kontrolu

Stejný model, stejné schopnosti, víc klávesnice. Jakmile si na desktop aplikaci zvykneš, přejdeš na CLI jedním příkazem:

```bash
curl -fsSL https://claude.ai/install.sh | sh
claude --version
```

Obojí sdílí stejný účet. Používej, co se hodí k úkolu.

## Účet a přihlášení

Potřebuješ Anthropic účet. [console.anthropic.com](https://console.anthropic.com) — přihlášení přes Google nebo mail.

**Před založením ověř svůj plán.** Claude Code funguje na osobních plánech (Pro, Max) i organizačních (Team, Enterprise). Pokud má tvoje firma pro tebe Team nebo Enterprise seat, použij ho — fakturace, retence dat i administrátorské nastavení se mezi osobními a organizačními tieri liší. Funkce jako **Remote Control** (ovládání pracovní stanice z prohlížeče nebo mobilu) vyžadují Pro nebo vyšší. Samotný API klíč na řadu funkcí nestačí.

Při prvním spuštění Claude otevře prohlížeč pro OAuth. To je normální. Tokeny se uloží do `~/.claude/` na macOS/Linux, do `%APPDATA%\Claude\` na Windows.

V prostředích, kde se prohlížečový callback nedostane na localhost — WSL2, SSH sessions, kontejnery — spusť `claude auth login` a OAuth kód vlož přímo do terminálu, až tě o to Claude požádá.

## Oprávnění

Claude se vždy zeptá, než něco provede. Výchozí chování:

- **Čtení souborů** — uvnitř projektu automaticky povoleno
- **Úprava souborů** — při první úpravě v dané konverzaci se zeptá
- **Spouštění příkazů** — při prvním spuštění v dané konverzaci se zeptá
- **Git akce** — zeptá se pokaždé

Jakmile pattern poznáš, přes `/permissions` si to přizpůsobíš. Pro tenhle lab nech výchozí hodnoty.

<Callout variant="tip">
Pokud jedeš na firemním stroji, ověř si to s IT před instalací. Některé firmy mají allow-list binárek a CLI instalátor ti může spadnout.
</Callout>

## Co opustí tvůj stroj

Každý prompt posílá:

- Tvoji zprávu
- Soubory, které jsi explicitně sdílel (přes `@mention`, tool use, nebo které si Claude přečetl, aby ti odpověděl)
- Historii konverzace v aktuálním kontextovém okně

**Neposílá:**

- Celý filesystem
- Soubory mimo adresář projektu
- Nic v `.gitignore` ani `.claudeignore`

### Retence, stručně

To, co obvykle zajímá firmy: jak dlouho si Anthropic drží, co jsi poslal?

- **Osobní plány (Pro, Max)** — standardní retence, používá se k vylepšování produktu; opt-out je v nastavení konzole.
- **Team / Enterprise** — nastavitelné adminem. Zero retention je dostupná na Enterprise. Politiku určuje tvůj admin, ty ji dědíš.
- **`.claudeignore`** — nejjednodušší obrana. Vedle `.gitignore`, vypíše soubory nebo cesty, které se ke Claudovi nikdy neposílají bez ohledu na plán. Používej pro tajné klíče, zákaznická data, proprietární kód.

Kapitola 9.5 má hlubší referenci s odkazy na stránky Anthropicu o zpracování dat a compliance. Pokud jedeš v týmu s konkrétními požadavky na compliance, poraď se s adminem *dřív*, než začneš lab.

## Minimální požadavky

- macOS, Windows nebo Linux
- Node 20+ pro CLI
- ~500 MB místa
- Internet
- Pro Remote Control a mobilní aplikaci: plán Pro nebo vyšší (kapitola 6)
