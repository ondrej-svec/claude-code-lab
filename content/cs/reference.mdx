## 8.1 Slash příkazy

V libovolné konverzaci s Claude Code napiš `/` a uvidíš kompletní seznam filtrovaný podle tvého plánu, platformy a nainstalovaných pluginů. Tohle jsou vestavěné příkazy, které má smysl znát.

### Session

| Příkaz | Co dělá |
|---|---|
| `/clear` | Nová konverzace. Stará se archivuje — obnovíš ji přes `/resume`. |
| `/compact [focus]` | Konverzace zůstává, Claude shrne historii a uvolní kontext. Volitelný focus zúží, co zachovat. |
| `/resume` | Navázat na předchozí konverzaci. Session na pozadí (spuštěné přes `--bg`) se tu teď taky zobrazují, označené jako `bg`. |
| `/rewind` | Checkpoint menu — obnoví kód, konverzaci nebo obojí do dřívějšího stavu. Stejné jako dvojité Esc. |
| `/branch` | Větvení konverzace od zadaného bodu. |
| `/exit` | Ukončí Claude Code (nebo `Ctrl+D`). |

### Konfigurace

| Příkaz | Co dělá |
|---|---|
| `/init` | Prohlédne repo a navrhne startovní `CLAUDE.md`. Nastav `CLAUDE_CODE_NEW_INIT=1` pro nový interaktivní multi-fázový flow. |
| `/memory` | Prohlédni nebo uprav CLAUDE.md soubory a načtenou paměť. |
| `/permissions` | Spravuj allow/deny pravidla pro nástroje. |
| `/model` | Přepne model pro aktuální session (např. Opus → Sonnet). Výběr modelu se teď automaticky ukládá jako výchozí pro nové session. Stiskni `s` v pickeru, jestli chceš přepnout jen pro tuhle session. `fallbackModel` v `settings.json` nastaví řetěz až tří záložních modelů, které se vyzkouší postupně, pokud je primární přetížený nebo nedostupný. `--fallback-model` platí i pro interaktivní session. |
| `/fast` | Přepne fast mode. Výchozí je teď Opus 4.8 — 2× standardní cena, 2,5× rychlost. |
| `/effort` | Per-session override effortu. |
| `/keybindings` | Otevře `~/.claude/keybindings.json`. |
| `/hooks` | Konfigurace lifecycle hooků. |
| `/config` | Otevře nastavení. |
| `/theme` | Přepni téma terminálu. |

### Info

| Příkaz | Co dělá |
|---|---|
| `/help` | Seznam příkazů. |
| `/powerup` | Interaktivní onboarding — deset animovaných průchodů po funkcích, které většina lidí přehlédne. |
| `/context` | Vizualizace využití kontextového okna s tipy na optimalizaci. |
| `/cost` | Dosavadní cena session. |
| `/usage`, `/stats` | Metriky využití. `/usage` teď zobrazí rozpad podle kategorií — skills, subagenty, pluginy a náklady per MCP server. |
| `/doctor` | Diagnostika — vypíše konflikty v konfiguraci, chybějící nastavení. |

### Workflow

| Příkaz | Co dělá |
|---|---|
| `/agents` | Správa subagentů (vytvořit, upravit, smazat). |
| `/tasks` | Správa úkolů na pozadí — rozjet dlouhý běh, výsledek vyzvednout později. |
| `/workflows` | Přehled tvých dynamických workflow. |

Přidej `--fix` a nálezy se rovnou aplikují do working tree. `/simplify` jede jen úklidové kolo — znovupoužití, zjednodušení, efektivita, výška abstrakce — ne plný lov chyb. |

## 8.2 Klávesové zkratky

Zkratky v CLI jsou kontextové. Kompletní seznam je na [code.claude.com/docs/en/keybindings](https://code.claude.com/docs/en/keybindings). Vlastní si uprav v `~/.claude/keybindings.json` přes `/keybindings`.

### Musíš znát

| Zkratka | Akce |
|---|---|
| `Esc Esc` | **Rewind** — obnoví se předchozí zpráva (stejné jako `/rewind`). |
| `Ctrl+C` | Přeruší aktuální vstup nebo odpověď. **Neukončuje Claude.** Rezervované. |
| `Ctrl+D` | Ukončí Claude Code. Rezervované. |
| `Shift+Tab` | Cyklí režim oprávnění: default → acceptEdits → plan → auto. |
| `Ctrl+R` | Reverse-search v historii — standardně prohledává všechny projekty. `Ctrl+S` zúží hledání na aktuální projekt nebo session. |
| `Ctrl+O` | Přepíná podrobný transkript (rozbalí zkrácené tool volání). |
| `Ctrl+T` | Přepíná seznam úkolů. |
| `Ctrl+B` | Úkol na pozadí. (Konfliktuje s tmux prefixem — zmáčkni dvakrát.) |
| `Ctrl+J` | Nový řádek bez odeslání. |
| `Ctrl+G` nebo `Ctrl+X Ctrl+E` | Otevře aktuální prompt v `$EDITOR`. |
| `Ctrl+V` (`Cmd+V` v iTerm2) | Vloží obrázek ze schránky jako `[Image #N]`. |
| `Cmd+P` / `Meta+P` | Výběr modelu. |
| `Cmd+T` / `Meta+T` | Přepne rozšířené myšlení. |
| `Shift+Enter` nebo `\+Enter` | Víceřádkový vstup. Nativně v iTerm2, WezTerm, Ghostty, Kitty, Warp, Apple Terminal. Jinde: `/terminal-setup`. |

### Gotcha

Option-klávesové zkratky na macOS tiše selhávají, dokud si v terminálu nenastavíš **Left/Right Option = Esc+** (iTerm2 atd.). Pokud `Alt+M` nedělá pro cyklení režimů nic, je to tohle — použij `Shift+Tab`.

Zkratky desktop aplikace Anthropic oficiálně nedokumentuje — to, co tady vidíš, je CLI.

## 8.3 Vstupní prefixy

Claude Code čte první znak řádku jako signál režimu:

/>

| První znak | Režim |
|---|---|
| `/` | Slash příkaz — spustí příkaz, ne prompt. |
| `!` | Bash passthrough — pustí shell, výsledek se dostane do tvého kontextu. |
| `@` | Zmínka souboru — `@path/to/file` připojí soubor; `@Explore` zmíní subagenta. |
| `#` | Memory add — text za `#` se připojí do správného `CLAUDE.md`. Claude vybere rozsah (projekt / uživatel / lokální). |
| Cokoliv jiného | Normální prompt. |

`#` použij, když se přistihneš, že opakovaně vysvětluješ stejné pravidlo. `!` použij, když chceš, aby Claude viděl výstup reálného příkazu. `@` použij, když chceš, aby Claude přečetl konkrétní soubor bez hádání.

## 8.4 CLAUDE.md — jak na to

Kanonická dokumentace paměti žije na [code.claude.com/docs/en/memory](https://code.claude.com/docs/en/memory). Praktický tvar:

### Cílová délka

**Pod 200 řádků na soubor.** Oficiální doporučení. Za tím bobtná spotřeba kontextu a Claude se pravidel drží méně spolehlivě. Komunitní konsenzus jde ještě dál — dobré CLAUDE.md soubory mají často pod 80 řádků.

### Co patří dovnitř

- **Co** projekt je (jeden odstavec: doména, velikost, stack)
- **Proč** je postavený takhle (jeden odstavec: architektura nebo omezení)
- **Jak** se na něm pracuje (příkazy, konvence, ne-dělej)

Multi-krokové postupy dej do skillů, ne do CLAUDE.md. Pravidla specifická pro cesty dej do `.claude/rules/*.md` s `paths:` frontmatter — aktivují se jenom, když jsou relevantní.

### Hierarchie rozsahů

Všechny `CLAUDE.md` soubory z aktuálního adresáře směrem k rootu se při startu session zřetězí. Priorita (pozdější vítězí při konfliktu):

1. **Managed** — politika od enterprise adminu (pokud existuje)
2. **User** — `~/.claude/CLAUDE.md` (o tobě, načítá se všude)
3. **Project** — `./CLAUDE.md` nebo `./.claude/CLAUDE.md` (o codebase, commitnuté)
4. **Local** — `./CLAUDE.local.md` (o tobě v tomto projektu, gitignorované)

`CLAUDE.md` v podadresářích se načítají líně, když Claude sáhne do souboru uvnitř. Užitečné pro monorepa.

### Tři šablony

**Projekt (`./CLAUDE.md`):**

```md
# Projekt

Interní onboarding nástroj. Postgres + Node, deploy na Vercel.

## Konvence
- TypeScript strict. Žádné `any` bez komentáře.
- Testy vedle zdrojáku: `foo.ts` + `foo.test.ts`.
- Migrace: číslované SQL soubory v `db/migrations/`.

## Ne
- Neinstaluj npm balíčky bez ptaní.
- Nesahej do `app/legacy/`.

## Příkazy
- `pnpm dev`, `pnpm test`, `pnpm db:seed`
```

**Uživatel (`~/.claude/CLAUDE.md`):**

```md
# Osobní

Preferuju přímé odpovědi. Žádné rozehřívání.
V editoru používám vim keybindings.
Když řeknu „ship it", nejdřív pusť testy, pak commit.
```

**Lokální (`./CLAUDE.local.md`, gitignorované):**

```md
# Lokální poznámky

Staging DB: `postgresql://localhost:5433/app_staging`
WIP branch: `wip/2026-04-refactor-auth`
```

### Importy

`@path/to/file.md` se rozbalí inline — užitečné, ale **importovaný soubor pořád čerpá z kontextu**, načítá se při startu session. Max hloubka 5 skoků. Nepoužívej importy na „úsporu kontextu" — neušetří.

### Co jde špatně

- **Přeinstrukování** — pravidla se uniformně ignorují
- **Osobnostní próza** — plýtvá tokeny bez zlepšení výstupu
- **Konfliktní pravidla ve vnořených souborech** — Claude si vybere jedno libovolně
- **Použité jako linter** — nech lintery lintovat; CLAUDE.md nastavuje záměr

## 8.5 Privacy, bezpečnost a plány

Odlehčený přehled. Aktuální privacy a trust stránky Anthropicu jsou zdrojem pravdy, odkazy jsou níže.

### Co opustí tvůj stroj

Každý prompt posílá: tvoji zprávu, soubory, které jsi explicitně sdílel (přes `@`, tool use, nebo které si Claude přečetl, aby odpověděl), a historii konverzace v aktuálním kontextovém okně.

**Neposílá:** celý filesystem, soubory mimo projekt, nic, co sedí na `.gitignore` nebo `.claudeignore`.

### Plány a retence

| Plán | Pro koho | Jak je na tom s retencí |
|---|---|---|
| **Pro** | Jednotlivec | Standardní retence; používá se k vylepšování služby; opt-out v nastavení konzole. |
| **Max** | Intenzivní jednotlivec | Stejné jako Pro. |
| **Team** | Malé organizace (5–200 seats) | Nastavitelné adminem. Přísnější defaulty než Pro. |
| **Enterprise** | Velké organizace | Zero retention k dispozici. Admin řídí politiku. SSO, SOC 2, DPA. |
| **API přímo** | Vývojáři, některé funkce blokované | Retenci si nastavuješ sám v API konzoli; Remote Control, mobilní aplikace a týmové funkce jsou obvykle mimo hru. |

Pokud má tvoje firma Team nebo Enterprise seat, **použij ho**. Rozdíly v retenci a admin politice jsou pro firemní použití důležité.

### `.claudeignore`

Nejjednodušší obrana bez ohledu na plán. Vedle `.gitignore`, vypisuje soubory nebo cesty, které Claude nikdy nevidí:

```
.env
.env.*
**/secrets.*
customer-data/
proprietary-algorithms/
```

Používej pro tajné klíče, zákaznická data, interní dokumenty — co ti diktuje compliance.

### Sandboxing

`/sandbox` obalí bash příkazy sandboxem na úrovni OS — filesystem + síť (Seatbelt na macOS, bubblewrap na Linuxu/WSL2). [Kapitola 8](/cs/lab/next-steps#sandboxing) to vysvětluje. Flagy, které jsou důležité pro spravovaná nasazení:

| Flag | Efekt |
|---|---|
| `sandbox.failIfUnavailable: true` | Tvrdý fail, když sandbox nestartuje. Žádný tichý fallback. |
| `allowUnsandboxedCommands: false` | Vypne escape hatch, přes který by jinak příkazy běžely mimo sandbox. |
| `sandbox.network.allowManagedDomainsOnly: true` | Na síť jen admin-schválené domény. |
| `sandbox.filesystem.allowWrite` | Cesty mimo CWD, kam můžou subprocesy zapisovat. Slučují se napříč scopy nastavení. |

→ [Sandboxing docs](https://code.claude.com/docs/en/sandboxing)


### MCP bezpečnost

MCPs, které načítají nedůvěryhodný obsah (webové stránky, issues, maily), do tvé session můžou injektovat instrukce. Postoj Anthropicu: **třetí strany na vlastní riziko**. Před instalací MCP si zkontroluj, co dělá. Upřednostňuj first-party MCPs od organizací `anthropics/`, když jsou k dispozici.

## 8.6 Jak Claude rozšiřovat

### Skilly

Agent Skills otevřený formát — specifikace a referenční implementace:

- [agentskills.io](https://agentskills.io) — otevřený standard
- [github.com/anthropics/skills](https://github.com/anthropics/skills) — 17 oficiálních skillů + meta-skilly `skill-creator` a `mcp-builder`
- [Skills docs](https://code.claude.com/docs/en/skills)

Kapitola 7 ukazuje agentní workflow psaní skillů (vyvoláš `skill-creator` nebo `craft-skill`, odpovíš na otázky objevování, zkontroluješ draft).

### Pluginy

Formát plugin manifestu, submission marketplace, rozdíl plugin vs. skill:

- [Plugins docs](https://code.claude.com/docs/en/plugins)
- [Plugins reference](https://code.claude.com/docs/en/plugins-reference)
- Odeslat plugin: [claude.ai/settings/plugins/submit](https://claude.ai/settings/plugins/submit)

### Hooky

Shell příkazy, HTTP endpointy nebo LLM prompty spouštěné na konkrétních lifecycle událostech. Události zahrnují `SessionStart`, `PreToolUse`, `PostToolUse`, `FileChanged`, `Stop`, `PreCompact`, `MessageDisplay` (transformuje nebo schová text asistenta při zobrazení) a řadu dalších.

- [Hooks docs](https://code.claude.com/docs/en/hooks)
- Konfigurace v `.claude/settings.json` pod `"hooks"`, nebo v `hooks/hooks.json` pluginu
- Hooky dostávají aktivní úroveň effortu session jako `effort.level` v JSON vstupu a jako `$CLAUDE_EFFORT` v prostředí — hodí se pro hooky, jejichž chování má záviset na aktuálním effortu

### MCPs

Postav si vlastní MCP server s MCP SDK (Python nebo TypeScript):

- [Model Context Protocol](https://modelcontextprotocol.io) — otevřený standard
- Skill `mcp-builder` v `anthropics/skills` ti postaví základ
- [Claude Code MCP docs](https://code.claude.com/docs/en/mcp)
