# Za oponou

Krátká kapitola na konci průvodce, který tě učil Claude Code — o tom, co stálo postavit *tento průvodce* právě v Claude Code.

Ve dnech 23.–24. dubna 2026, během šesti sezení a zhruba čtrnácti hodin doby, po kterou byla sezení otevřená, vývojář s agentem doručili všechno, co jsi právě četl: scaffold, devět kapitol ve dvou jazycích, ukázkové projekty, doprovodný skill, hero ilustraci, čtyři diagramy, testy, deploy. Dost z těch čtrnácti hodin byl agent pracující bez dozoru na schválených plánech — hodin, kdy byl člověk u toho aktivně, bylo spíš běžné odpoledne. Tohle jsou čísla pod tím, taháno přímo z lokálních transkriptů sezení Claude Code, oceněno v aktuálních API sazbách Anthropicu se správnými cache multiplikátory.

## Proč je tohle tady

Dva důvody.

**Transparentnost.** Když někdo čte návod k workshopu a ptá se *„jak dlouho to vlastně trvalo a co to stálo,"* upřímná odpověď patří přímo do návodu, ne do následného e-mailu. Čtrnáct hodin doby otevřených sezení, velká část bez dozoru. $845 v API sazbě. Nulový dodatečný výdaj na Max předplatném, na kterém ta práce skutečně běžela.

**Demonstrace.** Nejsilnější důkaz, že vzory z [kapitoly 4](/cs/lab/iteration-and-control) a [kapitoly 6](/cs/lab/compound-engineering) — plan mode, proof slices, compound loop, posture založená na harnessu — skutečně škálují, je lab postavený přesně v těch vzorech. Tahle kapitola je stvrzenka.

## Praxe: vyrenderuj si dashboard vlastního používání

Dobré závěrečné cvičení: udělej z toho, co jsi právě četl, dashboard své vlastní spotřeby. Vzory z kapitol 4 a 6 drží: úzký rozsah, napřed plán a pak práce, proof slice před dávkou.

**Zkus tenhle plan prompt** — jeden úzký úkol, jeden konkrétní výstup:

> `/plan`: Napiš mi Python skript, který přečte moje jsonl soubory sezení Claude Code z posledních 30 dní v `~/.claude/projects/` a vyrenderuje jednostránkový HTML dashboard. Pro každé sezení: délka, počet tahů, počet subagentů, součty tokenů, cena v API sazbě. Použij aktuální sazby z platform.claude.com. Jeden HTML soubor. Bez závislostí mimo Python stdlib.

Nech Claude napsat plán, přečti si ho, přitáhni rozsah, pokud se rozjel jinam, a pak `/work` proti schválenému plánu. Týden vlastních sezení vyrenderovaný jako dashboard ti řekne o tom, jak *ty* pracuješ s Claude Code, víc než jakákoli případová studie, kterou publikoval někdo jiný.
