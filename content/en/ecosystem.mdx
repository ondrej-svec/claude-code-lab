# The ecosystem

Claude Code alone is a good agent. With the ecosystem, it becomes a platform you extend instead of wait for.

This chapter is a tour, not a tutorial. Skim it, mark what matters to you, come back later for depth.

## Skills

A **skill** is a bundle of instructions + optional scripts that Claude loads on demand. A domain-specific playbook — lives in `~/.claude/skills/`, activates automatically when the situation matches.

**An example worth a look:** [`visualize`](https://github.com/ondrej-svec/heart-of-gold-toolkit). Takes any markdown document — a plan, a brainstorm, an architecture doc — and turns it into a polished, shareable HTML artifact. One well-defined job, well-scoped instructions, a few helper scripts. The shape of a good skill.

### Build your own

Three steps:

1. **Pick one specific job** you do repeatedly. Not "help with coding" — something like *"turn this plan into a shareable HTML artifact"* or *"write a release note in our voice."*
2. **Write a `SKILL.md`** with a small frontmatter block + clear instructions:

   ```md
   ---
   name: my-skill
   description: One sentence — when should Claude activate this?
   ---

   # My skill

   When asked to do X, do this:

   1. Read the input.
   2. Apply Y.
   3. Return Z.
   ```

3. **Drop it in** `~/.claude/skills/my-skill/`. Claude finds it next time around.

Start rough. Use it twice. Refine. That's the loop.

→ [Skills docs](https://docs.claude.com/en/docs/claude-code/skills)

## Plugins

A **plugin** bundles related skills, commands, and hooks into one installable unit. Install once, get the whole workflow.

Chapter 6 walks through installing and using the [Compound Engineering plugin](https://github.com/every-to/compound-engineering) — Avery's full brainstorm → plan → work → review → compound loop. That's the one to try first. [Super Powers](https://github.com/iannuttall/claude-super-powers) is another worth knowing about — a broader pick-and-choose toolkit.

As a reference point: the [Heart of Gold toolkit](https://github.com/ondrej-svec/heart-of-gold-toolkit) is a small collection of skills I bundled for my daily work (visualize is one of them). Not something to install now — an example of what emerges once you've written three or four skills you don't want to lose. You'll build one of your own.

→ [Plugins docs](https://docs.claude.com/en/docs/claude-code/plugins)

## MCPs

**Model Context Protocol** lets Claude talk to external services — filesystems, databases, browsers, APIs — as first-class tools.

One standout for web developers:

- **Chrome DevTools MCP** — Claude gets live access to Chrome's console, network, performance profiler. "Open the page, tell me why LCP is 3.2s, fix it." Claude can now actually look.

Other MCPs exist for Gmail, Google Drive, Linear, Notion, Postgres, and more. Install what matches your flow.

→ [MCP docs](https://docs.claude.com/en/docs/agents-and-tools/mcp)

## Subagents

When Claude spawns a task that would burn a lot of context — deep codebase exploration, running a big review — it uses a **subagent**. The subagent does its work in its own context window, then hands back a summary.

You don't usually invoke subagents directly. You just notice that your main conversation stays clean even when a heavy task runs in the background.

Why it matters: your context window is finite. Subagents keep it focused on the task you care about.

## Remote and mobile

Two ways to run Claude when you're not at your desk:

- **Remote control** — Claude Code runs on your workstation, you drive it from a browser or phone. Useful for long-running jobs you kick off from a meeting.
- **Mobile app** — lighter, for quick reviews, reading conversation history, approving waiting tasks.

Sign in with the same account; pick up mid-session.

The ecosystem is a menu, not a checklist. Install what actually saves you time. Everything else can wait.
