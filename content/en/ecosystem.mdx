# The ecosystem

Claude Code alone is a good agent. With the ecosystem, it becomes a platform you extend instead of wait for.

This chapter is a tour, not a tutorial. Skim it, mark what matters to you, come back later for depth.

## Skills

A **skill** is a bundle of instructions + files + optional scripts that Claude can load on demand. Think of it as a domain-specific playbook.

**A good example: `visualize`** from the [Heart of Gold toolkit](https://github.com/ondrej-svec/heart-of-gold-toolkit). Takes any markdown document — a plan, a brainstorm, an architecture doc — and turns it into a polished, shareable HTML artifact. One well-defined job, done well. The skill I reach for most.

Other skills worth installing from the toolkit:

- `write-post` — guided long-form writing, end-to-end
- `frontend-design` — production-grade UI generation without the generic AI look
- `rclone` — cloud storage sync patterns
- `clean-code-tdd` — refactoring and test-driven patterns
- `copy-editor` — typography audit and voice review

Install the whole toolkit or pick individual skills. See the [Heart of Gold toolkit install docs](https://github.com/ondrej-svec/heart-of-gold-toolkit#install).

The bigger point: these are exactly the kind of thing **you can build for yourself**. When you catch yourself running the same instructions three times, stop and write a skill. Chapter 6 is the how.

Skills live in `~/.claude/skills/` or come bundled in **plugins** (below). When a skill matches what you're doing, Claude invokes it automatically.

→ [Skills docs](https://docs.claude.com/en/docs/claude-code/skills)

## Plugins

A **plugin** is a bundle of skills, commands, and hooks — distributed as a unit. You install a plugin once and get all its capabilities.

Two worth installing this week:

- **[Compound Engineering](https://github.com/every-to/compound-engineering)** by Avery — the `brainstorm → plan → work → review → compound` loop as a full workflow. Ships with slash commands and reviewer agents.
- **[Super Powers](https://github.com/iannuttall/claude-super-powers)** — a broader toolkit. Pick-and-choose skills for daily work.

Install a plugin:

```
/plugin install <source>
```

→ [Plugins docs](https://docs.claude.com/en/docs/claude-code/plugins)

## MCPs

**Model Context Protocol** lets Claude talk to external services — filesystems, databases, browsers, APIs — as first-class tools.

One standout for web developers:

- **Chrome DevTools MCP** — Claude gets live access to Chrome's console, network, performance profiler. "Open the page, tell me why LCP is 3.2s, fix it." Claude can now actually look.

Other MCPs exist for Gmail, Google Drive, Linear, Notion, Postgres, and more. Install what matches your flow.

→ [MCP docs](https://docs.claude.com/en/docs/agents-and-tools/mcp)

## Subagents

When Claude spawns a task that would burn a lot of context — deep codebase exploration, running a big review — it uses a **subagent**. The subagent does its work in its own context window, then hands back a summary.

You don't usually invoke subagents directly. You just notice that your main conversation stays clean even when a heavy task runs in the background.

Why it matters: your context window is finite. Subagents keep it focused on the task you care about.

## Remote and mobile

Two ways to run Claude when you're not at your desk:

- **Remote control** — Claude Code runs on your workstation, you drive it from a browser or phone. Useful for long-running jobs you kick off from a meeting.
- **Mobile app** — lighter, for quick reviews, reading conversation history, approving waiting tasks.

Sign in with the same account; pick up mid-session.

The ecosystem is a menu, not a checklist. Install what actually saves you time. Everything else can wait.
